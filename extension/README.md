# Page Watcher

Page Watcher는 현재 열려 있는 일반 웹사이트에서 사용자가 직접 선택한 영역을 주기적으로 확인하고, 이전 snapshot과 달라지면 Chrome 알림과 모바일 Web Push 알림을 보내는 Manifest V3 Chrome Extension입니다.

## 주요 기능

- 일반 `http://` 및 `https://` 웹페이지에서 감시 영역 선택
- 드래그로 표 셀처럼 여러 감시 영역을 한 번에 등록
- 검사 전에 클릭할 새로고침 버튼/링크 액션 등록
- 페이지 내부 플로팅 패널 제공
- 수동 확인
- Chrome alarms 기반 주기적 확인
- 선택 영역 변경 알림과 변경 영역 라벨 표시
- 드래그 선택 중 후보 DOM 영역 표시
- 현재 감시 중인 DOM 영역 상시 표시
- QR 기반 모바일 Web Push 등록
- 감시 설정 삭제

## 보안 원칙

- 사이트 계정/비밀번호를 저장하지 않습니다.
- 감시 영역 snapshot과 자동 생성된 등록 채널은 `chrome.storage.local`에만 저장합니다.
- 알림 본문에 실제 감시 영역 값을 노출하지 않습니다.
- 자동 로그인이나 세션 유지 우회 기능을 제공하지 않습니다.

## 설치

1. Chrome에서 `chrome://extensions`를 엽니다.
2. 우측 상단의 개발자 모드를 켭니다.
3. `압축해제된 확장 프로그램을 로드`를 클릭합니다.
4. `extension` 폴더를 선택합니다.

## Push 서버

확장은 고정 Push 서버 `https://watcher.kangbeen.my`를 사용합니다. 엔드유저는 별도로 `npm install`이나 서버 실행을 할 필요가 없습니다.

운영자는 `push-server` 폴더의 서버를 Railway 같은 Node 서버 호스팅에 배포하고, `watcher.kangbeen.my`를 해당 서비스에 연결하면 됩니다. GitHub Pages는 정적 호스팅이라 Push 구독 저장과 `/notify` 발송 API를 처리할 수 없습니다. Vercel에 배포하려면 파일 저장 대신 KV/DB 저장소를 붙이는 방식으로 바꿔야 합니다.

Railway 배포 요약:

1. 이 저장소를 GitHub에 push합니다.
2. Railway에서 새 프로젝트를 만들고 GitHub 저장소를 연결합니다.
3. 서비스의 root directory를 `push-server`로 설정합니다.
4. start command는 `npm start`를 사용합니다.
5. Volume을 만들고 mount path를 `/app/data`로 설정합니다.
6. 환경변수 `DATA_DIR=/app/data`를 추가합니다.
7. Railway Public Networking 또는 Domains 설정에서 custom domain `watcher.kangbeen.my`를 추가합니다.
8. DNS 제공자에서 Railway가 안내하는 CNAME 또는 DNS 레코드를 `watcher.kangbeen.my`에 설정합니다.
9. `https://watcher.kangbeen.my/register.html`이 열리는지 확인합니다.

## 사용법

1. 감시할 웹페이지를 엽니다.
2. Chrome 툴바에서 Page Watcher 확장 아이콘을 클릭합니다.
3. `핸드폰 등록 QR 표시`를 누릅니다.
4. 휴대폰 Chrome으로 QR을 열고 `이 기기에 알림 등록`을 누른 뒤 알림을 허용합니다.
5. 데스크톱 플로팅 패널에서 `푸시 테스트`로 수신을 확인합니다.
6. `감시 영역 선택/드래그`를 누릅니다.
7. 단일 영역은 클릭하고, 표처럼 여러 영역은 사각형으로 드래그합니다.
8. 필요하면 `새로고침 액션 등록`을 누르고 검사 전에 클릭할 버튼이나 링크를 선택합니다.
9. `지금 확인`으로 수동 검사를 실행합니다.
10. 검사 주기를 선택하고 `주기 저장`을 누르면 대상 페이지가 열려 있을 때 주기적으로 검사합니다.

## 플로팅 패널

Chrome의 기본 확장 popup이나 별도 popup window를 사용하지 않습니다. 확장 아이콘을 누르면 현재 웹페이지 안에 플로팅 패널을 주입합니다. 페이지를 클릭해도 패널은 유지되며, 패널의 `x` 버튼으로 닫을 수 있습니다.

감시 영역을 드래그하는 동안 선택 후보 DOM은 주황색 점선으로 표시됩니다. 저장된 감시 영역은 초록색 테두리로 계속 표시됩니다. 같은 탭에서 새 감시 영역을 설정하면 기존 감시 설정은 덮어쓰기됩니다.

## Web Push 방식

QR은 `https://watcher.kangbeen.my/register.html?channel=...` 등록 페이지를 가리킵니다. 휴대폰에서 이 페이지를 열어 알림 권한을 허용하면 서버가 해당 기기의 Push subscription을 채널에 저장합니다. 이후 확장이 변경을 감지하면 Push 서버의 `/notify`로 요청하고, 서버가 구독된 기기에 Web Push 알림을 보냅니다.

`감시 설정 삭제`를 누르면 확장은 로컬 감시 설정을 지우기 전에 `https://watcher.kangbeen.my/channel/{channel}`로 삭제 요청을 보내 서버에 저장된 해당 채널의 구독도 제거합니다.

감시 설정은 다음 경우에도 자동으로 삭제됩니다.

- 감시 영역을 설정한 Chrome 탭이 닫힌 경우
- 감시 설정이 생성된 지 7일이 지난 경우

자동 삭제 시 로컬 감시 설정과 서버 채널 구독이 함께 정리됩니다.

## 한계

- Chrome 내부 페이지, Chrome Web Store, 일부 브라우저 보호 페이지에는 content script를 주입할 수 없습니다.
- 대상 페이지가 열려 있을 때만 주기 검사를 수행합니다.
- cross-origin iframe 내부 DOM에는 접근할 수 없습니다.
- 선택 영역의 텍스트를 비교하며, 사이트별 데이터 구조를 파싱하지 않습니다.
- 드래그 선택은 표 셀(`td`, `th`)을 우선 등록하고, 표가 아니면 드래그 사각형 안의 텍스트 요소를 등록합니다.
- 모바일 Web Push 등록에는 `watcher.kangbeen.my`가 HTTPS로 정상 배포되어 있어야 합니다.
- 등록 채널은 확장이 자동 생성합니다. 이 채널이 포함된 QR/등록 링크를 공유하면 다른 기기도 같은 알림을 받을 수 있습니다.

## 테스트 시나리오

1. 일반 웹페이지에서 확장 아이콘을 눌러 플로팅 패널이 뜨는지 확인합니다.
2. QR이 자동 생성된 채널로 표시되는지 확인합니다.
3. 휴대폰 Chrome에서 QR 등록 후 `푸시 테스트` 알림이 도착하는지 확인합니다.
4. `감시 영역 선택/드래그` 후 hover 오버레이와 selector 저장을 확인합니다.
5. 드래그 중 후보 DOM 영역이 주황색 점선으로 표시되는지 확인합니다.
6. 저장 후 현재 감시 영역이 초록색 테두리로 계속 표시되는지 확인합니다.
7. `새로고침 액션 등록` 후 버튼/링크 selector 저장을 확인합니다.
8. `지금 확인` 최초 실행 시 알림 없이 snapshot만 저장되는지 확인합니다.
9. 테스트용 DOM 텍스트를 바꾼 뒤 `지금 확인`을 눌러 변경 알림을 확인합니다.
10. 감시 대상이 사라졌을 때 변경 알림이 아니라 패널 상태 메시지만 표시되는지 확인합니다.
11. 같은 탭에서 새 감시 영역을 설정하면 기존 표시/설정이 덮어써지는지 확인합니다.
12. `감시 설정 삭제` 후 Push 서버의 해당 채널 구독이 삭제되는지 확인합니다.
13. 감시 영역을 설정한 탭을 닫으면 로컬 설정과 서버 채널 구독이 삭제되는지 확인합니다.
